### ¿Qué es un flujo de datos?
Data Pipeline es la representación del viaje de los datos, desde su punto inicial (crudo) hasta convertirse en una gráfica o reporte que permitan tomar decisiones.
* Es importante ya que el flujo de trabajo nos garantiza tres cosas:
    1. **_Integridad:_** Garantizamos que el dato es veridico y no se perdió en el camino.
    1. **_Automatización:_** No tenemos que descargar manualmente cada archivo cada semana, el flujo lo hace por nosotros.
    1. **_Escalabilidad:_** Nos permite manejar millones de filas con la misma facilidad con la que manejamos diez.
        *  Usamos **_código o reglas lógicas_** (SQL, Python, herramientas ETL) 
        * Escalabilidad Vertical vs. Horizontal
## Datos en Movimientos y Datos Estaticos
### 1. Datos Estáticos (Data at Rest)
Son datos físicos o digitales, no estan activamente viajando por la red en este momento. Se analizan mediante **Batch** (Lotes) "Hacia atrás" mira lo que ya paso, ventas del mes pasado.
### 1. Datos en Movimiento (Data in Motion)
Son conocidos como **Data in Flight** o **Streaming** son datos que se mueven activamente por una red, estos se generan continuamente a partir de interacciones del negocio, eventos del sistema o integraciones entre aplicaciones. Estos datos deben ser capturados, procesados y almacenados mientras se están produciendo.
* **Enfoque del analista:** Se analizan en Tiempo Real. Aquí el valor está en la inmediatez. Como analista, podrías crear tableros que se actualizan cada segundo para detectar anomalías.
* **Seguridad:** El foco esta en el cifrado del canal de transmisión (como protocolos TLS/SSl) para que nadie intercepte la tubería.
### Tabla de comparación
| Característica | Datos Estáticos (En Reposo / At Rest) | Datos en Movimiento (En Tránsito / In Motion) |
|---------------|----------------------------------------|----------------------------------------------|
| Definición | Datos almacenados de forma persistente en un soporte físico o digital. | Datos que fluyen continuamente desde una fuente a un destino a través de una red. |
| Volumen | Generalmente masivo (históricos acumulados de años). | Variable; el flujo es constante pero el procesamiento suele ser por eventos individuales. |
| Velocidad de Procesamiento | Batch (Lotes): Se procesan en intervalos programados (ej. cada noche). | Streaming: Procesamiento inmediato en tiempo real o casi real. |
| Herramientas de Análisis | SQL, Power BI, Tableau, Python (Pandas/Polars), Excel. | Apache Kafka, Spark Streaming, Flink, AWS Kinesis, Azure Stream Analytics. |
| Latencia | Alta: El tiempo desde que ocurre el evento hasta que se analiza puede ser de horas o días. | Baja: El análisis ocurre milisegundos después de que se genera el dato. |
| Tipo de Análisis | Descriptivo/Diagnóstico: Enfocado en reportes históricos y tendencias pasadas. | Predictivo/Prescriptivo: Enfocado en alertas inmediatas y modelos de detección al vuelo. |
| Arquitectura Típica | Data Warehouse (BigQuery, Snowflake) o Data Lakes. | Message Brokers (Kafka) y Motores de Procesamiento de Eventos. |
| Estado del Dato | Datos "Pasivos": Están esperando a ser consultados. | Datos "Activos": Están disparando acciones o alertas mientras se mueven. |
| Seguridad | Cifrado de archivos, control de acceso a bases de datos y seguridad física. | Cifrado de canales (SSL/TLS) y seguridad en los puntos finales (endpoints). |
| Ejemplo de Uso | Informe trimestral de ventas para la junta directiva. | Detección de una transacción fraudulenta antes de que sea aprobada. |
## Origen de los datos
Existen diversas fuentes para obtener datos:
1. **OLTP:** Estos sistemas están diseñados para registrar operaciones del negocio, como ventas,
registros de usuarios o transacciones financieras. En ellos, los datos se crean y actualizan
constantemente
1. **APIs:** Son un conjunton de reglas y protocolos, estan permiten que dos aplicaciones se comuniquen entre si. Como analistas estas nos sirven como ventanilla para traer datos específicos y la API no los entrega en formato estructurado (JSON o XML)
1. **LOGS y EVENTOS:** representan registros generados automáticamente por los sistemas. Estos pueden incluir errores, actividades de usuarios o eventos específicos dentro de una aplicación, y son especialmente relevantes en arquitecturas modernas.
