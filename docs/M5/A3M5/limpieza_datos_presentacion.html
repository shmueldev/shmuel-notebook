Técnicas SQL
Consultas clave para limpieza
1 — Detectar nulos y vacíos
SELECT COUNT(*) AS total_nulos FROM clientes WHERE telefono IS NULL OR TRIM(telefono) = '';
2 — Eliminar duplicados (CTE + ROW_NUMBER)
WITH duplicados AS ( SELECT id, ROW_NUMBER() OVER ( PARTITION BY email ORDER BY id ) AS rn FROM clientes ) DELETE FROM clientes WHERE id IN (SELECT id FROM duplicados WHERE rn > 1);
3 — Normalizar texto (nombre, ciudad)
UPDATE clientes SET nombre = INITCAP(TRIM(nombre)), ciudad = INITCAP(LOWER(TRIM(ciudad)));
Desafíos avanzados
Deduplicación y outliers
Estrategias de deduplicación
  • 1Exacta: DISTINCT o GROUP BY sobre campos únicos.
  • 2Fuzzy matching: similitud con Levenshtein o Jaro-Winkler.
  • 3Record linkage: cruzar IDs entre sistemas distintos.
  • 4Blocking: agrupar primero para reducir comparaciones.
Detección de outliers
  • AZ-score: valores a más de 3 desviaciones estándar.
  • BIQR: fuera del rango intercuartil extendido.
  • CReglas de negocio: edad > 120, precio negativo.
  • DAcción: marcar, imputar o eliminar según contexto.
⚠ Precaución: No eliminar outliers sin validar con el negocio. Pueden ser señal de fraude o evento real.
Buenas prácticas
Principios para datos limpios
🛡 Prevención
  • Constraints en la BD
  • Validación en la app
  • Estándares de entrada
  • Catálogos de referencia
⚙ Proceso
  • Limpiar en staging
  • Versionar con dbt
  • Documentar decisiones
  • Log de cambios
📡 Monitoreo
  • Tests post-carga
  • Métricas continuas
  • Alertas por umbrales
  • Data contracts
Regla de oro: La limpieza de datos es un proceso continuo, no un evento único. Automatiza, monitorea y documenta cada decisión.
Resumen final
Conclusiones
  • 1Los datos sucios generan pérdidas económicas y decisiones incorrectas.
  • 2Pipeline: Auditar → Corregir → Deduplicar → Validar → Monitorear.
  • 3SQL es la base; ETL y dbt escalan el proceso a millones de registros.
  • 4Prevenir con constraints es siempre más barato que corregir millones de registros.
  • 5La calidad de datos es responsabilidad de todo el equipo, no solo del DBA.
